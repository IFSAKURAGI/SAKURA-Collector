# SAKURA 收集组件

高性能 Figma 组件收集插件，具备智能扫描和优化的组件管理功能。

## 📋 功能特性

### 扫描模式
- **整个文件**：扫描当前 Figma 文件中的所有组件
- **当前页面**：仅扫描当前页面内的组件
- **当前选择**：仅扫描选中范围内的组件

### 组件管理
- 自动统计主组件、变体、实例数量
- 支持按全部 / 外部 / 内部组件筛选
- 点击列表项可定位到对应母组件（外部组件定位到首个实例）
- 区分组件与变体，显示来源（本地 / 外部文件）

### 收集功能
- 将主组件收集到指定页面
- 自动网格布局排列
- 实例自动重绑到新页面主组件
- 支持「移动当前文件组件」「外部组件单独页面」等选项

### 界面
- State A：扫描前 - 选择扫描范围
- State B：扫描后 - 组件列表与统计
- State C：设置弹窗 - 配置收集选项
- State D：进度中 - 实时进度与取消
- State E/F：成功 / 失败反馈

## 🚀 快速开始

### 开发环境

```bash
# 安装依赖
npm install

# 编译 TypeScript
npm run build

# 监听模式
npm run watch
```

### 导入插件

1. 打开 Figma 桌面应用
2. `Plugins` → `Development` → `Import plugin from manifest...`
3. 选择项目中的 `manifest.json`

## 📖 使用方法

1. **扫描**：点击「整个文件」「当前页面 / 当前选择」开始扫描
2. **筛选**：使用右上角下拉筛选全部 / 外部 / 内部组件
3. **定位**：点击列表中的组件名称，在 Figma 中定位到对应节点
4. **收集**：点击「收集」进入设置，配置目标页面和选项后执行 GO

## ⚙️ 配置

目标页面默认名称：`📦 Components Library`

布局与性能参数可在 `code.ts` 中调整。

## 🐛 故障排除

- **扫描无结果**：确认文件/页面/选区中存在组件实例
- **实例无法重绑**：部分外部组件可能因权限限制无法重绑
- **插件未响应**：大型文件扫描需较长时间，可查看进度条状态
